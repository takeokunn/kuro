# Basic Usage and Key Bindings

このチュートリアルでは、kuro での日常的なターミナル操作方法と、よく使うキーバインドについて学びます。

## ターミナルの基本操作

### 新しいターミナルセッションの開始

```elisp
M-x kuro
```

または、特定のシェルを指定して起動：

```elisp
M-x kuro RET /bin/zsh
```

### ターミナルセッションの終了

```bash
# シェル側から終了
exit

# または Emacs のキーバインドで
C-c C-c  ; kuro-quit
```

## キーバインド一覧

### 基本的なキーバインド

| キーバインド | コマンド | 説明 |
|-------------|----------|------|
| `C-c C-c` | `kuro-quit` | ターミナルセッションを終了 |
| `C-c C-z` | `kuro-suspend` | ターミナルを一時停止 |
| `C-c C-o` | `kuro-clear-scrollback` | スクロールバックをクリア |
| `C-c C-l` | `kuro-redraw` | 画面を再描画 |

### コピーペースト

| キーバインド | コマンド | 説明 |
|-------------|----------|------|
| `C-c C-p` | `kuro-paste` | クリップボードからペースト |
| `C-RET` | `kuro-send-input` | 入力を送信 |

### ウィンドウ操作

| キーバインド | コマンド | 説明 |
|-------------|----------|------|
| `C-x 1` | `delete-other-windows` | 他のウィンドウを閉じてターミナルのみ表示 |
| `C-x 2` | `split-window-below` | ウィンドウを上下に分割 |
| `C-x 3` | `split-window-right` | ウィンドウを左右に分割 |

## 実践的な使用例

### 例 1: Git 操作

```bash
# リポジトリの状態を確認
git status

# 変更をステージング
git add .

# コミット
git commit -m "Add new feature"

# プッシュ
git push
```

### 例 2: ファイル操作

```bash
# ディレクトリを移動
cd ~/projects

# ファイルの一覧を表示
ls -la

# 新しいディレクトリを作成
mkdir new-project

# ファイルを編集 (vim などのターミナルエディタ)
vim README.md
```

### 例 3: プロセス管理

```bash
# 実行中のプロセスを確認
ps aux

# 特定のプロセスを検索
ps aux | grep nginx

# プロセスを終了
kill 1234
```

## kuro 固有の機能

### スクロールバックの操作

kuro は効率的なスクロールバック管理を実装しています：

```bash
# 長い出力を生成
yes | head -n 1000

# スクロールバックを遡る
M-x kuro-scroll-up

# スクロールバックを戻る
M-x kuro-scroll-down

# スクロールバックを検索
M-x kuro-search-scrollback
```

### 複数のターミナルセッション

複数のターミナルセッションを同時に実行できます：

```elisp
# 最初のターミナル
M-x kuro RET kuro-1

# 2番目のターミナル
C-x 3 M-x kuro RET kuro-2
```

バッファ名を指定することで、複数のセッションを管理できます。

### アルテate スクリーン対応

`less` や `vim` などのアルテate スクリーンを使用するプログラムもシームレスに動作します：

```bash
# ページャーを使用
less /var/log/syslog

# テキストエディタを起動
vim config.txt

# less や vim を終了すると元の画面に戻ります
```

## パフォーマンスの最適化

### 大量の出力 handling

```bash
# AI エージェントからの大量出力などもスムーズに処理
yes | head -n 100000

# ログファイルの監視
tail -f /var/log/nginx/access.log
```

kuro は Dirty Line Tracking を使用しているため、差分のみを更新し、高速な表示を実現しています。

## トラブルシューティング

### 画面が乱れる場合

```elisp
M-x kuro-redraw
```

で画面を再描画できます。

### 入力が反映されない場合

```bash
# ターミナル側で Enter を押してみる
# または Emacs 側で
C-g  ; キーボードクォートを解除
```

### カーソル位置がおかしい場合

```bash
# シェル側でリセット
reset

# または Emacs 側で
M-x kuro-reset-cursor
```
